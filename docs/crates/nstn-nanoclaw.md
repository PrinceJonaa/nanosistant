## Purpose

`nstn-nanoclaw` is the **edge tier** of Nanosistant. It is the first point of contact for all user messages and operates with a strict priority:

1. **Resolve locally** — call `try_deterministic_resolution()` from `nstn-common`. Zero network, zero tokens.
2. **Forward via gRPC** — if the message cannot be resolved locally and a gRPC connection to RuFlo exists, forward it.
3. **Queue offline** — if the network is unavailable, enqueue the message and return a queued notice.

The edge tier is intentionally thin. All complex routing, agent dispatch, and memory management live in `nstn-ruflo`.

### `EdgeRuntime`

```rust
pub struct EdgeRuntime { /* private */ }

impl EdgeRuntime {
    /// Create a runtime in offline-only mode (local deterministic + queue).
    pub fn new() -> Self;

    /// Create a runtime with a gRPC client for forwarding to RuFlo.
    pub fn with_grpc(client: GrpcClient) -> Self;

    /// Process a message. Returns immediately for deterministic hits;
    /// otherwise forwards to RuFlo or enqueues if offline.
    pub async fn process(
        &mut self,
        request: EdgeRequest,
        context: &mut SessionContext,
    ) -> Result<EdgeResponse, EdgeError>;

    /// Access the offline queue.
    pub fn queue(&self) -> &OfflineQueue;

    /// Drain the offline queue and attempt to forward all messages.
    /// Returns the number of messages successfully forwarded.
    pub async fn flush_queue(&mut self, context: &mut SessionContext) -> usize;
}
```

**Processing order:**
1. `LocalExecutor::try_resolve(&request.message)` — if `Some(answer)`, return `EdgeResponse::local(answer)`.
2. If a `GrpcClient` is present and connected → call `grpc_client.send(&request).await`.
3. If gRPC fails or no client → `queue.enqueue(&request.message, &request.domain_hint)` and return a queued notice.

### `GrpcClient`

```rust
pub struct GrpcClient { /* private */ }

impl GrpcClient {
    /// Create a client targeting `endpoint`. Starts disconnected.
    pub fn new(endpoint: &str) -> Self;

    /// Establish a tonic channel to the endpoint.
    /// Returns GrpcError::Transport if the connection fails.
    pub async fn connect(&mut self) -> Result<(), GrpcError>;

    pub fn is_connected(&self) -> bool;

    /// Forward an EdgeRequest to RuFlo and return its EdgeResponse.
    /// Returns GrpcError::NotConnected when not yet connected.
    pub async fn send(&mut self, request: &EdgeRequest) -> Result<EdgeResponse, GrpcError>;
}
```

**Wire protocol:** Uses the `NanoClawServiceClient` generated by `tonic` from `nanosistant.proto`. The `EdgeRequest` is mapped to the proto `EdgeRequest` type (with `user_message` from `request.message`). The proto `EdgeResponse` is mapped back to the edge `EdgeResponse`.

## Module: `local`

**File:** `crates/nanoclaw/src/local.rs`

```rust
pub struct LocalExecutor;

impl LocalExecutor {
    /// Try to resolve `message` via the deterministic function library.
    /// Returns Some(answer) for closed-form queries, None otherwise.
    pub fn try_resolve(message: &str) -> Option<String>;
}
```

This is a thin delegation to `nstn_common::try_deterministic_resolution`. The `LocalExecutor` struct exists to provide a clean module boundary and make the responsibility explicit.

**Examples that resolve locally:**

```
"c major scale"           → "C major scale: C - D - E - F - G - A - B"
"140 bpm bar duration"    → "At 140 BPM (4/4): one bar = 1.714s"
"what time is it"         → "Current time: 2026-04-02T13:26:00Z"
"Am in C major"           → "Am in C major = vi"
"2500hz band"             → "2500 Hz → Upper Mids"
"word count: hello world" → "2 words"
```

**Examples that return None** (require LLM judgment):

```
"help me write a verse about love"
"what's the best investment right now"
"explain recursion"
```

## Module: `sync`

**File:** `crates/nanoclaw/src/sync.rs`

In-memory offline queue. Messages are held here when the gRPC connection to RuFlo is unavailable and can be drained once connectivity is restored.

```rust
pub struct QueuedMessage {
    pub message: String,
    pub domain_hint: String,
    pub timestamp: String,   // ISO 8601 / RFC 3339
}

pub struct OfflineQueue { /* private */ }

impl OfflineQueue {
    pub fn new() -> Self;

    /// Push a message onto the back of the queue.
    pub fn enqueue(&mut self, message: &str, domain_hint: &str);

    /// Remove and return all queued messages in FIFO order.
    pub fn drain(&mut self) -> Vec<QueuedMessage>;

    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
}
```

**Note:** The queue is in-memory only and does not survive process restarts. For durable queueing across restarts, persist the `QueuedMessage` list to disk before shutdown and reload on startup.

## Usage Example

```rust
use nstn_nanoclaw::{EdgeRuntime, EdgeRequest, GrpcClient, SessionContext};

// Offline mode: deterministic resolution + queueing
let mut runtime = EdgeRuntime::new();
let mut ctx = SessionContext::new();

let response = runtime.process(
    EdgeRequest {
        message: "c major scale".into(),
        domain_hint: String::new(),
        session_id: "session-1".into(),
    },
    &mut ctx,
).await.unwrap();

assert!(response.from_local);
assert!(response.content.contains("C - D - E - F - G - A - B"));

// Connected mode: forward to RuFlo
let mut client = GrpcClient::new("http://localhost:50051");
client.connect().await.unwrap();

let mut runtime = EdgeRuntime::with_grpc(client);
let response = runtime.process(
    EdgeRequest {
        message: "help me write a verse".into(),
        domain_hint: "music".into(),
        session_id: "session-2".into(),
    },
    &mut ctx,
).await.unwrap();

println!("{}", response.content);

// Low-level: check offline queue
let local = nstn_nanoclaw::LocalExecutor::try_resolve("140 bpm bar duration");
assert!(local.is_some());
```

## Relationship to Other Crates

```
nstn-nanoclaw
    ├── calls nstn-common::try_deterministic_resolution (via local::LocalExecutor)
    ├── generates nstn-common::proto::EdgeRequest for gRPC
    └── receives nstn-common::proto::EdgeResponse from RuFlo (nstn-ruflo::NanoClawGrpcService)
```

`nstn-nanoclaw` has no dependency on `nstn-ruflo` at the Rust type level — the two crates communicate exclusively through the protobuf wire format defined in `nstn-common::proto`.
